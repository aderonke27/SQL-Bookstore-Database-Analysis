## Project Overview
This project focuses on analyzing an online bookstore database using SQL queries within Python. The project was completed as part of my AltSchool Data Analysis second semester program and involved extracting meaningful business insights from a relational database. The analysis explored customer behavior, book sales performance, revenue generation, and review patterns by querying interconnected database tables. This project demonstrates practical SQL skills, relational database understanding, and analytical thinking applied to a real-world business scenario.

## Problem Statement
As the database administrator for an online bookstore, the task was to write SQL queries to retrieve and analyze important business information stored across multiple tables.

Each analytical question was treated as an individual SQL problem and solved using structured queries.

## Dataset and Database Structure
The project used a relational bookstore database that contained multiple interconnected tables already provided within the notebook environment. The database included the following tables:
* Authors
* Books
* Customers
* Orders
* Order_Items
* Reviews

These tables stored information related to:
* Books and their authors
* Customer details
* Customer orders and purchased items
* Book reviews and ratings

The database and setup codes were preloaded as part of the project notebook. The task focused mainly on writing SQL queries to access, retrieve, and analyze data from the provided tables in order to answer the given business questions.

## Methodology
The project was completed by following the provided notebook instructions and writing SQL queries to solve the given analytical tasks. The steps involved were:
* Imported the required libraries and enabled the SQL extension in Jupyter Notebook using the pre-provided setup code
* Connected to the SQLite bookstore database
* Explored the database tables to understand the available data and relationships
* Wrote SQL queries to answer each analytical question
* Applied SQL techniques such as joins, filtering, aggregation, grouping, and sorting while solving the tasks
* Validated the query results by comparing them with the expected outputs provided in the notebook
* Compiled the SQL queries, outputs, and findings into a documented report

## Questions Answered
* Retrieve a list of the top 5 most popular books based on the number of times they have been ordered. Also order by author name in descending order
* Calculate the total revenue generated by each author from the sales of their books
* Identify the top 5 customers who have placed the most orders. Also order by customer name in descending order
* Find all orders that contain book with book_id = 1
* Select detailed review information of the Least 5 Books based on ratings, including book title, author name, review rating, and review text

## Results and Insights
* The analysis showed that **1984 by George Orwell** was the most frequently ordered book, with a total of 4 orders, while books like *The Hobbit*, *Pride and Prejudice*, *Adventures of Huckleberry Finn*, and *Harry Potter and the Philosopher’s Stone* also ranked among the top ordered books.
* Revenue analysis revealed that **George Orwell generated the highest total revenue (256.83)** from book sales, followed by J.K. Rowling and Jane Austen, indicating stronger customer demand for their books.
* Customer order analysis showed that **David Wilson and Alice Johnson placed the highest number of orders**, with 6 orders each, demonstrating the most active purchasing behavior among customers in the database.
* Order tracking for **book_id = 1** identified multiple purchases made by customers such as Alice Johnson, Eva Martinez, and Charlie Brown across different order dates, showing repeated demand for the selected book.
* Review analysis revealed that books like **1984** and **The Hobbit** received high ratings and positive review comments, indicating strong customer satisfaction and favorable reader experiences.
