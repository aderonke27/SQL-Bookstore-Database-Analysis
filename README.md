# Project Overview
This project focuses on analyzing an online bookstore database using SQL queries within Python. The project was completed as part of my AltSchool Data Analysis second semester program and involved extracting meaningful business insights from a relational database.
The analysis explored customer behavior, book sales performance, revenue generation, and review patterns by querying interconnected database tables.
This project demonstrates practical SQL skills, relational database understanding, and analytical thinking applied to a real-world business scenario.

# Problem Statement
As the database administrator for an online bookstore, the task was to write SQL queries to retrieve and analyze important business information stored across multiple tables.
The objective was to:
* Analyze book popularity and sales
* Calculate author revenue
* Evaluate customer purchasing behavior
* Retrieve order details
* Explore customer review patterns
* Generate meaningful business insights using SQL
Each analytical question was treated as an individual SQL problem and solved using structured queries.

# Methodology
The project was carried out using the following steps:
1. Loaded the required libraries and SQL extension in Jupyter Notebook
2. Connected to the SQLite bookstore database
3. Explored the database tables and relationships
4. Wrote SQL queries to answer each analytical question
5. Used SQL joins, aggregation functions, filtering, grouping, and sorting techniques
6. Validated outputs against expected results
7. Compiled the analysis and outputs into a documented report

# Key Questions Answered
### Book Sales Analysis
* Retrieve the top 5 most popular books based on the number of times they were ordered
* Calculate total revenue generated by each author

### Customer Analysis
* Identify the top 5 customers with the highest number of orders

### Order Analysis
* Find all orders containing a specific book (`book_id = 1`)

### Review Analysis
* Retrieve detailed review information for books, including:
  * Book title
  * Author name
  * Rating
  * Review text

# Results and Insights
* Certain books emerged as significantly more popular based on customer order frequency.
* Authors generated different levels of revenue depending on book demand and pricing.
* Customer purchasing behavior revealed that some customers placed considerably more orders than others.
* Popular books contributed heavily to the bookstore’s total revenue.
* Revenue analysis showed how book pricing and purchase quantity impact author earnings.
* Some customers demonstrated strong engagement through repeated purchases.
* Order tracking provided insights into which books appeared most frequently in customer transactions.
* Review analysis showed generally positive customer feedback and high ratings for popular books.
* Highly rated books often aligned with the most frequently purchased titles.
* SQL joins made it possible to combine data from multiple related tables for deeper business analysis.
